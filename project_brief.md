# StockSquad — Project Brief
**A Multi-Agent Stock Research System**

---

## Overview

StockSquad is a Python-based multi-agent system that collaborates to analyze stocks and produce investment theses. A squad of specialized AI agents — each with a distinct role and memory — debate, research, and synthesize information to deliver a structured report on any given ticker.

The primary goal is **learning**: mastering multi-agent orchestration, ML-based signal scoring, vector memory, and agent coordination patterns using **Azure AI Foundry** as the backbone.

---

## Goals

### Learning Objectives
- Understand multi-agent orchestration and inter-agent communication
- Work hands-on with vector databases and semantic memory (short-term vs long-term)
- Build and train a lightweight ML scoring model on time-series financial data
- Apply NLP and embeddings for sentiment analysis
- Gain practical experience with Azure AI Foundry (agent management, tool use, evaluation)

### Product Objective
A working CLI (and optionally simple UI) where you input a stock ticker and receive a structured research report generated by the agent squad — including a bull/bear debate, a technical signal score, and a final recommendation.

---

## The Agent Squad

| Agent | Role | Key Tools |
|---|---|---|
| **OrchestratorAgent** | Manages the workflow, assigns tasks, synthesizes the final report | Azure AI Foundry orchestration |
| **DataAgent** | Fetches price history, volume, financial statements | Yahoo Finance / Alpha Vantage API |
| **TechnicalAgent** | Computes RSI, MACD, moving averages; runs ML signal scoring | scikit-learn, pandas, ta-lib |
| **SentimentAgent** | Scrapes news & earnings transcripts, classifies tone | NLP embeddings, HuggingFace |
| **FundamentalsAgent** | Analyzes P/E, revenue growth, margins, debt ratios | Financial data APIs |
| **DevilsAdvocateAgent** | Actively challenges the prevailing bull/bear case | LLM reasoning |

---

## Memory Architecture

Each agent has access to two memory layers:

### Short-Term Memory (Session)
- In-context message history for the current analysis run
- Shared "scratchpad" where agents post intermediate findings
- Managed via Azure AI Foundry's thread/context management

### Long-Term Memory (Persistent)
- Vector store (ChromaDB or Azure AI Search) indexed by ticker + date
- Agents retrieve past analyses when re-analyzing the same stock
- Example: *"Last time we analyzed NVDA (3 months ago), we underweighted the data center narrative — flag this."*
- Embeddings model: `text-embedding-ada-002` via Azure OpenAI

---

## ML Component

### Signal Scoring Model
- **Input features**: RSI, MACD crossover, volume delta, 20/50/200 day MA relationship, earnings surprise history
- **Target**: Price direction 30 days forward (binary: up/down, or tertile: up/flat/down)
- **Model**: Start with `RandomForestClassifier` (scikit-learn), optionally upgrade to LightGBM
- **Training data**: Historical data for S&P 500 stocks (Yahoo Finance, 5+ years)
- **Output**: Confidence-weighted signal fed into TechnicalAgent's report

### Sentiment Scoring
- Embed news headlines and earnings call snippets using `sentence-transformers`
- Cluster by theme (growth, risk, guidance, macro) using k-means
- Track narrative shift over time: compare current embedding centroid to 3-month average

## Project Phases

### Phase 1 — Foundation (Weeks 1–2)
**Goal: Two agents talking, basic memory working**

- [ ] Set up Azure AI Foundry project and agent environment
- [ ] Build `DataAgent`: fetches OHLCV + basic financials for a ticker
- [ ] Build `OrchestratorAgent`: assigns task, collects DataAgent output
- [ ] Implement short-term shared scratchpad (in-context)
- [ ] Implement basic long-term memory: store and retrieve past runs in ChromaDB
- [ ] CLI: `python main.py analyze --ticker AAPL`

**Deliverable**: A CLI that fetches stock data and stores the result in vector memory, retrievable on the next run.

---

### Phase 2 — The Squad (Weeks 3–5)
**Goal: Full agent squad with ML scoring**

- [ ] Build `TechnicalAgent` with TA indicators (RSI, MACD, MAs)
- [ ] Train Signal Scoring Model on historical data
- [ ] Integrate model output into TechnicalAgent's report
- [ ] Build `SentimentAgent` with news scraping + embedding-based scoring
- [ ] Build `FundamentalsAgent` with key ratio analysis
- [ ] Build `DevilsAdvocateAgent` that challenges the consensus
- [ ] Wire all agents through Orchestrator with structured message passing
- [ ] Long-term memory: agents retrieve and reference past analyses

**Deliverable**: Full squad produces a multi-section report for any ticker.

---

### Phase 3 — Polish & Learning Wrap-up (Weeks 6–8)
**Goal: Evaluation, backtesting, and optional UI**

- [ ] Implement backtesting: run squad on historical dates, measure accuracy of final recommendation vs actual price movement
- [ ] Use Azure AI Foundry Evaluation to score agent output quality
- [ ] Add Streamlit UI: ticker input → rendered report with charts
- [ ] Refine memory: add memory decay (older analyses weighted lower)
- [ ] Document learnings: write up what worked, what didn't, what you'd do differently

**Deliverable**: A polished, demo-able system with evaluation metrics and a backtest result.
